## Get your first checkout running

This guide walks you through creating a Polar account, setting up your first product, and generating a checkout link. No code required to get started.

<Steps>
  <Step title="Create your Polar account">
    Head to [polar.sh](https://polar.sh) and sign up with your GitHub account.
    
    <Note>
      Polar uses GitHub OAuth for authentication. This also enables automatic GitHub repository access benefits if you need them later.
    </Note>
    
    After signing in, you'll create your first organization. This represents your business or project.
  </Step>

  <Step title="Create your first product">
    Navigate to **Products** in the dashboard sidebar and click **Create Product**.
    
    <Tabs>
      <Tab title="Subscription">
        For recurring revenue (SaaS, memberships):
        
        1. **Name**: "Pro Plan" 
        2. **Description**: Describe what customers get
        3. **Type**: Select **Subscription**
        4. **Price**: Set your monthly price (e.g., $29/month)
        5. **Billing Interval**: Choose Monthly or Yearly
        
        <Tip>
          You can add both monthly and yearly prices to the same product by creating multiple price points.
        </Tip>
      </Tab>
      
      <Tab title="One-Time">
        For single purchases (ebooks, courses, tools):
        
        1. **Name**: "Starter Kit"
        2. **Description**: What's included in the purchase
        3. **Type**: Select **One-time**
        4. **Price**: Set your price (e.g., $49)
        
        <Note>
          One-time products don't require ongoing subscription management.
        </Note>
      </Tab>
      
      <Tab title="Pay-What-You-Want">
        Let customers choose their price:
        
        1. **Name**: "Support the Project"
        2. **Description**: Thank supporters
        3. **Type**: Select **One-time**
        4. **Price Type**: Choose **Pay what you want**
        5. **Minimum**: Set minimum amount (e.g., $5)
      </Tab>
    </Tabs>
    
    Click **Create Product** to save.
  </Step>

  <Step title="Add benefits (optional)">
    Benefits are what customers receive after purchasing. You can add them now or skip this step.
    
    <AccordionGroup>
      <Accordion title="GitHub Repository Access">
        Automatically invite customers to private repositories:
        
        1. Go to your product page
        2. Click **Add Benefit** → **GitHub Repository Access**
        3. Connect your GitHub account if not already connected
        4. Select the repository
        5. Choose permission level (Read, Write, or Admin)
        
        <Warning>
          You must be an owner or admin of the repository to grant access.
        </Warning>
      </Accordion>
      
      <Accordion title="Discord Role">
        Auto-assign Discord roles to customers:
        
        1. Click **Add Benefit** → **Discord**
        2. Connect your Discord server
        3. Select the role to assign
        4. Customers will receive an invite link after purchase
      </Accordion>
      
      <Accordion title="License Key">
        Generate unique license keys:
        
        1. Click **Add Benefit** → **License Keys**
        2. Configure key settings:
           - Prefix for your keys
           - Expiration (optional)
           - Activation limit (optional)
        3. Keys are generated automatically on purchase
      </Accordion>
      
      <Accordion title="File Downloads">
        Provide secure download links:
        
        1. Click **Add Benefit** → **File Downloads**
        2. Upload your files (PDFs, ZIPs, etc.)
        3. Files are available in the customer portal
      </Accordion>
    </AccordionGroup>
    
    [Learn more about Benefits →](/features/benefits)
  </Step>

  <Step title="Set up Stripe (for production)">
    Polar uses Stripe to process payments. For testing, you can skip this step.
    
    <Tabs>
      <Tab title="Test Mode">
        By default, you're in **Sandbox mode**. All checkouts use Stripe test mode:
        
        - Use test card: `4242 4242 4242 4242`
        - Any future expiry date
        - Any CVC code
        
        Perfect for testing your integration before going live.
        
        [Learn more about Sandbox →](/integrate/sandbox)
      </Tab>
      
      <Tab title="Production">
        To accept real payments:
        
        1. Go to **Settings** → **Finance** → **Accounts**
        2. Click **Connect Stripe**
        3. Complete Stripe onboarding:
           - Business information
           - Banking details for payouts
           - Tax information
        4. Polar will verify your account (usually instant)
        
        <Warning>
          Stripe onboarding is required to accept real payments. Polar cannot process payments without a connected Stripe account.
        </Warning>
        
        [View supported countries →](/merchant-of-record/supported-countries)
      </Tab>
    </Tabs>
  </Step>

  <Step title="Create a checkout link">
    Now create a shareable link customers can use to purchase:
    
    1. Navigate to **Checkout Links** in the sidebar
    2. Click **Create Checkout Link**
    3. Configure your checkout:
       
       <ParamField path="Product" type="required">
         Select the product you created
       </ParamField>
       
       <ParamField path="Success URL" type="optional">
         Where to redirect customers after payment:
         ```
         https://yoursite.com/welcome?checkout_id={CHECKOUT_ID}
         ```
         The `{CHECKOUT_ID}` parameter is replaced automatically.
       </ParamField>
       
       <ParamField path="Label" type="optional">
         Internal label to organize your links (e.g., "Twitter Bio Link")
       </ParamField>
       
       <ParamField path="Allow Discount Codes" type="boolean">
         Let customers enter discount codes at checkout
       </ParamField>
    
    4. Click **Create** to generate your link
    
    You'll get a URL like:
    ```
    https://polar.sh/your-org/checkout/link-123
    ```
    
    Share this link anywhere - website, social media, emails, etc.
  </Step>

  <Step title="Test your checkout">
    Open your checkout link in a new tab to test the flow:
    
    1. Click through the checkout process
    2. In test mode, use: `4242 4242 4242 4242`
    3. Complete the test purchase
    4. Check **Orders** in your dashboard to see the test order
    5. If you added benefits, verify they were granted in **Customers**
    
    <Tip>
      The customer portal is available at `https://polar.sh/customer-portal`. Customers can manage subscriptions, download invoices, and access benefits.
    </Tip>
  </Step>
</Steps>

## Common customizations

### Add a discount code

1. Go to **Discounts** in the sidebar
2. Click **Create Discount**
3. Configure:
   - **Code**: `LAUNCH20` (customer enters this)
   - **Type**: Percentage or fixed amount
   - **Value**: 20% or $10 off
   - **Duration**: Forever, once, or repeating
4. Customers can enter this code at checkout

[Learn more about Discounts →](/features/discounts)

### Offer a free trial

1. Edit your subscription product
2. Scroll to **Trial Period**
3. Set trial duration (e.g., 14 days)
4. Choose whether to require a payment method

Customers get full access during the trial. Billing starts automatically after the trial ends.

[Learn more about Trials →](/features/trials)

### Embed checkout in your site

Instead of redirecting to Polar's hosted checkout, embed it:

<CodeGroup>
```typescript Next.js
import { Polar } from "@polar-sh/sdk";

const polar = new Polar({
  accessToken: process.env.POLAR_ACCESS_TOKEN,
});

// Create checkout session
const checkout = await polar.checkouts.create({
  productPriceId: "prp_...",
  successUrl: "https://yoursite.com/success",
});

// Redirect to checkout.url
return Response.redirect(checkout.url);
```

```html JavaScript
<script src="https://cdn.polar.sh/checkout.js"></script>
<script>
  const checkout = new PolarCheckout({
    checkoutId: "checkout_...",
  });
  
  checkout.mount("#checkout-container");
</script>
```
</CodeGroup>

[View full embedding guide →](/guides/checkout-session)

## API-first workflow

Prefer to manage everything via API? Here's the complete flow:

<Steps>
  <Step title="Get an access token">
    Create a Personal Access Token or Organization Access Token:
    
    1. Go to **Settings** → **Tokens**
    2. Click **Create Token**
    3. Select scopes: `products:write`, `checkouts:write`
    4. Copy the token (shown once)
    
    [Learn more about Authentication →](/integrate/authentication)
  </Step>
  
  <Step title="Create a product">
    <CodeGroup>
    ```typescript TypeScript
    const product = await polar.products.create({
      name: "Pro Plan",
      description: "Full access to all features",
      prices: [
        {
          amountType: "fixed",
          priceAmount: 2900, // $29.00 in cents
          priceCurrency: "USD",
          recurringInterval: "month",
        },
      ],
    });
    ```
    
    ```python Python
    product = polar.products.create(
        name="Pro Plan",
        description="Full access to all features",
        prices=[
            {
                "amount_type": "fixed",
                "price_amount": 2900,  # $29.00 in cents
                "price_currency": "USD",
                "recurring_interval": "month",
            },
        ],
    )
    ```
    </CodeGroup>
  </Step>
  
  <Step title="Create a checkout">
    <CodeGroup>
    ```typescript TypeScript
    const checkout = await polar.checkouts.create({
      productPriceId: product.prices[0].id,
      successUrl: "https://yoursite.com/success",
      customerEmail: "customer@example.com",
    });
    
    // Redirect customer to checkout.url
    ```
    
    ```python Python
    checkout = polar.checkouts.create(
        product_price_id=product.prices[0].id,
        success_url="https://yoursite.com/success",
        customer_email="customer@example.com",
    )
    
    # Redirect customer to checkout.url
    ```
    </CodeGroup>
  </Step>
  
  <Step title="Listen for webhooks">
    Set up a webhook endpoint to receive events:
    
    ```typescript
    // POST /webhooks/polar
    export async function POST(req: Request) {
      const event = await req.json();
      
      if (event.type === "checkout.completed") {
        // Payment successful!
        const customerId = event.data.customer_id;
        const orderId = event.data.order_id;
        
        // Grant access in your system
        await grantAccess(customerId);
      }
      
      return new Response("OK", { status: 200 });
    }
    ```
    
    [View all webhook events →](/integrate/webhooks/events)
  </Step>
</Steps>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Checkout link doesn't work">
    **Issue**: Link shows "Product not found" or similar error
    
    **Solutions**:
    - Verify the product is not archived
    - Ensure you're not in Sandbox mode if using production link
    - Check that the product has at least one active price
  </Accordion>
  
  <Accordion title="Benefits not granted">
    **Issue**: Customer completed checkout but didn't receive benefits
    
    **Solutions**:
    - Check **Orders** → select the order → view **Benefits** tab
    - Look for errors in the benefit grant status
    - For GitHub: Ensure the repository is accessible
    - For Discord: Verify bot has proper permissions
    - Check webhook logs for `benefit.grant.created` events
  </Accordion>
  
  <Accordion title="Stripe connection fails">
    **Issue**: Can't complete Stripe onboarding
    
    **Solutions**:
    - Verify your country is supported ([view list](/merchant-of-record/supported-countries))
    - Ensure you have a valid business entity
    - Check Stripe's [restricted businesses](https://stripe.com/legal/restricted-businesses)
    - Contact [support](mailto:support@polar.sh) if issues persist
  </Accordion>
  
  <Accordion title="Test payments failing">
    **Issue**: Test checkouts show payment errors
    
    **Solutions**:
    - Use Stripe test card: `4242 4242 4242 4242`
    - Ensure you're in Sandbox mode (check dashboard banner)
    - Try a different test card from [Stripe's test cards](https://stripe.com/docs/testing)
    - Check browser console for errors
  </Accordion>
</AccordionGroup>
