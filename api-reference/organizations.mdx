## Overview

Organizations are the top-level entities in Polar. They own products, customers, and all other resources.

## The Organization Object

  Unique identifier
</ResponseField>

<ResponseField name="name" type="string" required>

## List Organizations

```bash cURL
curl -X GET "https://api.polar.sh/v1/organizations" \
  -H "Authorization: Bearer polar_pat_..."
```

### Query Parameters

<ParamField query="slug" type="string">
  Filter by slug
</ParamField>

## Get Organization

```bash cURL
curl -X GET "https://api.polar.sh/v1/organizations/{id}" \
  -H "Authorization: Bearer polar_pat_..."
```

### Path Parameters

<ParamField path="id" type="string" required>
  Organization ID or slug
</ParamField>

---

## Create Organization

```bash cURL
curl -X POST "https://api.polar.sh/v1/organizations" \
  -H "Authorization: Bearer polar_pat_..." \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Acme Corp",
    "slug": "acme"
  }'
```

### Request Body

<ParamField body="name" type="string" required>
  Organization name
</ParamField>

<ParamField body="slug" type="string" required>
  URL-friendly slug
</ParamField>

---

## Update Organization

```bash cURL
curl -X PATCH "https://api.polar.sh/v1/organizations/{id}" \
  -H "Authorization: Bearer polar_pat_..." \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Acme Corporation",
    "email": "support@acme.com"
  }'
```

### Path Parameters

<ParamField path="id" type="string" required>
  Organization ID
</ParamField>

### Request Body

All fields optional.

<ParamField body="name" type="string">
  Update name
</ParamField>

<ParamField body="email" type="string">
  Update contact email
</ParamField>

<ParamField body="billing_email" type="string">
  Update billing email
</ParamField>

<ParamField body="bio" type="string">
  Update bio
</ParamField>
